# Label

A form label element used to improve accessibility when pairing with form controls. Wraps a native `<label>` element with proper styling.

<DocsLink href="https://ui.shadcn.com/docs/components/label" />

<LabelBasicDemo client:load />

```tsx
import { Label } from "~/components/label";
import { Input } from "~/components/input";

export function LabelDemo() {
  return (
    <div class="grid w-full max-w-sm items-center gap-1.5">
      <Label for="label-name">Name</Label>
      <Input id="label-name" placeholder="Enter your name" />
    </div>
  );
}
```

## Installation

### CLI

Run the following command to add the component to your project:

```bash
npx solidui-cli@latest add label
```

### Manual

<div class="space-y-3">
Install the dependency:

```bash
npm install tailwind-variants
```
</div>

<div class="space-y-3">
Create the recipe file at `src/components/recipes/label.ts`:

```tsx
import { tv, type VariantProps } from 'tailwind-variants'

export const labelVariants = tv({
  base: "text-sm font-medium leading-none peer-disabled:cursor-not-allowed peer-disabled:opacity-70",
  variants: {
    error: {
      true: "text-destructive",
    },
  },
  defaultVariants: {
    error: false,
  },
})

export type LabelVariants = VariantProps<typeof labelVariants>
```
</div>

<div class="space-y-3">
Create the component file at `src/components/label.tsx`:

```tsx
import { splitProps, type Component } from 'solid-js'
import { labelVariants } from '../recipes/label'
import { ark, type HTMLArkProps } from '@ark-ui/solid/factory'

const Label: Component<HTMLArkProps<"label"> & { error?: boolean }> = (props) => {
  const [local, others] = splitProps(props, ["class", "error"])
  return <ark.label class={labelVariants({ class: local.class, error: local.error })} {...others} />
}

export { Label, labelVariants }
```
</div>

## Usage

Import the component:

```tsx
import { Label } from "@ui/solid";
```

The `for` attribute must match the `id` of the associated input element for proper accessibility and click-to-focus behavior:

```tsx
<Label for="email">Email</Label>
<Input id="email" type="email" />
```

### With Checkbox

Use the `ids` prop on Checkbox to set the hidden input's `id`, then pair it with Label's `for` attribute:

<LabelCheckboxDemo client:load />

```tsx
import { Label } from "~/components/label";
import { Checkbox } from "~/components/checkbox";

<Checkbox ids={{ hiddenInput: "terms" }} />
<Label for="terms">Accept terms and conditions</Label>
```

The Checkbox component passes the `ids` prop through to the underlying Zag.js machine, where `hiddenInput` controls the ID of the native `<input>` element. The Label's `for` attribute then references this ID for proper click-to-focus.

## API Reference

| Prop | Type | Default |
|------|------|---------|
| class | `string` | — |
| error | `boolean` | `false` |
| children | `any` | — |
| for | `string` | — |

All native `<label>` attributes are supported. The `for` attribute must match the `id` of the associated form element for click-to-focus to work.
